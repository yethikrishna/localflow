# iFlow

## Architecture Patterns
- **Workflow Automation Engine:** Focused on the "flow" of development rather than just the "code."
- **MCP-Centric:** Heavily relies on the Model Context Protocol to link local tools, system scripts, and enterprise suites.
- **Modular Core:** The core engine provides orchestration, context management, and tool execution, while specific logic is offloaded to sub-agents.
- **Terminal-First:** Optimized for CLI usage and automation pipelines.

## Key Features
- **iFlow CLI:** A powerful terminal assistant that serves as the entry point for all workflows.
- **IFLOW.md Memory:** A dedicated file-based memory system that developers can configure to improve AI collaboration and context.
- **Open Marketplace:** Allows users to add and share new SubAgents and MCP tools.
- **Enterprise Linkage:** Specifically designed to bridge the gap between local development tools and large enterprise software suites.

## How They Handle Agents
- **SubAgent Framework:** Supports a modular agent architecture where users can "spawn" specialized sub-agents for specific tasks.
- **MCP Integration:** Agents are "tooled" using MCP, allowing them to perform actions across a wide variety of services.
- **Context Management:** Focuses on maintaining a consistent "workflow context" through its memory system.

## LLM Support
- **Multi-LLM Strategy:** Supports various models including Claude, Gemini, and local models via MCP and CLI configuration.

## Unique Innovations & Differentiators
- **Workflow-to-Code:** Prioritizes automating the entire development workflow (requirements -> design -> implementation -> backup) over just code generation.
- **Vibe-Coding CLI Toolkit:** Positioned as the "Vibecoder's CLI," catering to the new wave of developers who leverage heavy AI automation.
- **Open Marketplace for Agents:** Unlike closed ecosystems, iFlow encourages a community-driven approach to agent and tool development.

# blink.so (Blink.new)

## Architecture Patterns
- **Full-Stack Agentic Platform:** Architecture encompasses not just code generation but also the infrastructure (hosting, database, authentication) required to run it.
- **Code-First Terminal Framework:** The Blink CLI (by Coder) provides a terminal-native way to build and deploy agents.
- **Vibe Coding Engine:** Optimized for rapid, chat-based application creation ("vibe coding").
- **MCP Enabled:** Supports the Model Context Protocol to give the agent access to external tools and APIs.

## Key Features
- **Blink.new:** A web-based platform to "chat an app into existence" in minutes.
- **Instant Infrastructure:** Automatically provisions databases, hosting, and auth for generated apps.
- **Blink CLI:** A fast way to build and deploy AI agents directly from the terminal.
- **Vibe Coding Blog/Resources:** A leading voice in the "vibe coding" movement, providing tools and guides for this new development style.

## How They Handle Agents
- **Agentic App Builder:** The platform itself is an agent that builds other agent-enabled apps.
- **Secure Execution Environments:** Blink runs agents inside secure cloud development environments (powered by Coder).
- **Tool Integration:** Uses MCP to allow agents to interact with backend code, databases, and third-party APIs.

## LLM Support
- **Model-Agnostic:** Supports multiple high-end models, with the ability for developers to choose the best model for their agent's specific needs.

## Unique Innovations & Differentiators
- **"Vibe Coding" Leadership:** Blink is a primary proponent and tool-builder for the "vibe coding" philosophy (high-level, intent-driven software creation).
- **Zero-to-Shipped Speed:** Focuses on reducing the time from "idea" to "hosted application" to mere minutes.
- **Unified Code + Infra:** Unlike Cursor or Claude Code which just write code, Blink aims to be the platform that also *runs* and *manages* the resulting software.
